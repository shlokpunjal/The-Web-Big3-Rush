# Day 4: JS Fundamentals

The purpose is to understand core JavaScript: variables, types, objects, arrow functions and use them to dynamically build a UI component.

## What does this project cover?
A Dev Card whose content is entirely built by JavaScript at runtime
- `const`, `let` and why `var` is avoided
- Data types and the `==` vs `===` trap
- Template literals and string interpolation
- Objects as data containers (no class needed)
- Arrow functions
- `.map()` to transform arrays
- `.join()` to collapse arrays into strings
- `innerHTML` to inject HTML from JS into the DOM

## Constraints
- No hardcoded content in HTML, just one empty `<div id="card">`
- All card content generated by JS on load
- No frameworks, no libraries

## Learnings
- `===` checks value AND datatype. `==` just checks value loosely, so always use `===`
- `typeof null` returns `"object"` — a historical JS bug frozen in time
- JS objects are just key-value pairs, no class required (basically a Python dict that can hold functions)
- JS has direct objects, no class needed to instantiate, the `class` syntax is just syntactic sugar


## How it works
```
devData object        →  holds all card content (name, bio, skills, links)
buildCard() called    →  constructs an HTML string using template literals
${devData.skills.map(...).join("")}  →  transforms skills array into span elements
card.innerHTML = ...  →  browser parses the string and renders real HTML elements
```

## File structure
```
dev-card/
├── index.html    (one empty div, links to CSS and JS)
├── style.css    (card layout, header/body split, hover states)
└── script.js     (devData object + buildCard function)
```
