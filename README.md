# The Web Big3 Rush

A 10-day sprint to go from HTML/CSS Foundations to writing a real code using the OG Javascript

## The Story

In the last semester of my diploma, HTML & CSS were part of the curriculum (MSBTE - CM2K). I learned them, used them, and got comfortable. When a 2-hour web design competition came around, I shifted my gears and started this repo to sharpen the CSS and pick up enough JS to make things interactive.
I just completed till Day 03 that time.

Fortunately, turns out that I didn't even need the JS. 
While everyone else rushed to add features, (few lacked CSS, let alone JS), I was the only one who actually pushed the CSS. Clean layout, real visual polish, deliberate design choices. That was the differentiator. I won with pure HTML and CSS. The Website - [Social Media Effects](https://shlokpunjal.github.io/Social-Media-Effects/)

But the sprint wasn't wasted, it reminded me that fundamentals done well beat half-baked complexity every time. And now, with the competition in the past, I'm finishing what I started. Days 4 through 10 are about genuinely understanding JavaScript — not for a deadline, just because the web deserves to be built properly.

## Structure

Each folder is one day of focused work:

- Day 01 – HTML Structure
- Day 02 – CSS Layout (Flexbox)
- Day 03 – Responsive Design
- Day 04 – JS Fundamentals
- Day 05 – DOM Manipulation
- Day 06 – Events + Local Storage
- Day 07 – Fetch API + Async JS
- Day 08 – CSS Animations + JS Orchestration
- Day 09 – Mock Build
- Day 10 – Capstone

## What's Different from Days 04 to 10
- The first three days were revision with a focus on doing things right: semantic HTML, no layout hacks, understanding why flexbox and grid exist. Days 4 onwards follow the same philosophy but for JS.
- Background in Java and Python means the syntax isn't the hard part. What's different in JS is the browser environment: the DOM, event-driven thinking, and how JS talks to a live webpage. That's the real learning here. No frameworks, no libraries — just vanilla JS that actually does things on screen.
